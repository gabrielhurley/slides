Horizon Project Update

October 2012

Horizon...

  • is the OpenStack Dashboard project
  • was first included as a Core project with the Essex release
  • has ~50 contributors from a diverse set of companies
  • is the ultimate consumer of every OpenStack API (we feel the pain)

Looking back...

 

“Think simple” as my old master used to say - meaning reduce the whole of its parts into the simplest terms, getting back to first principles.

—Frank Lloyd Wright

First principles

  • Core Support: Out-of-the-box support for all core OpenStack projects.
  • Extensible: Anyone can add a new component as a “first-class citizen”.
  • Manageable: The core codebase should be simple and easy-to-navigate.
  • Consistent: Visual and interaction paradigms are maintained throughout.
  • Stable: A reliable API with an emphasis on backwards-compatibility.
  • Usable: Providing an awesome interface that people want to use.

So how did we do?

Core Support

  • Full support for Nova, Swift, Glance, Keystone, Cinder
  • Added support for Quantum, will continue to improve (session thursday)

Extensible

  • Transformed project from thousands of lines of copy-and-paste one-offs to a framework
  • Quantum support was added entirely by Quantum team
  • Heat and Ceilometer are both exploring Horizon integration
  • More work to be done, but we've come a long way

Manageable

  • Logical structure for dashboards and panels
  • First commits in Grizzly cleaned up separation between framework "building blocks" and code for specific dashboards

Stable

  • Folsom release is fully backwards-compatible with Essex stack
  • Very few changes for developers
  • Minimal settings changes for deployers to upgrade

Usable

  • New "Workflows" in Folsom are a big step in the right direction
  • We can do better!

What does it mean?

We present the capabilities of the OpenStack platform
(Useful but limited)

Right now OpenStack Core is primarily IaaS—the dashboard reflects that

We provide a platform for ecosystem projects

  • At the Essex summit we heard a lot of people asking "how do I add my project to Horizon?"
    • We've answered that question in spades
  • People are working on integration
  • We're not the bottleneck or the arbiter of "who gets included"
  • We offer guidance to projects and try to build common features they need

The (Near) Future

Grizzly Themes

  • Making CRUD trivial (DRY like the desert)
  • More user-focused workflows
  • Great file upload interactions (swift, glance)
  • Great networking

Future++

Ramping up the curve

  • New projects appearing more rapidly, providing exciting new capabilities
    • Ceilometer, Heat, Synapse, CQS/CNS... the list goes on
  • Constructing the most useful integrated workflows is key.

PaaS, Saas...

  • Competing nascent solutions (CloudFoundry, Heat, OpenShift, etc.)
  • No standardization/consensus yet
    • There doesn't have to be one winner if they're interoperable
  • We'll support it the moment the community gets there
  • This is the future, and it's an exciting one!

Useful for diverse audiences (flexibly)

  • Basic users: Obvious workflows, Up-and-running fast vs. lots of choices
  • Project administrators: Supporting end users in their project, Monitoring and managing resources
  • Cloud administrators: Overall system utilization, Cloud-scale management interfaces
  • "Executives": Simple overviews, Solid reporting

More capabilities at your fingertips

  • Improved console/virtual desktop experience
  • Better visualizations
  • Real-time event-driven interaction
  • Integration with new and existing projects and services that people want to use

Challenges

  • Cross-project commonality for API features, data structures, etc.
  • Cross-project event-driven notification system
  • Design
    • Most common problem of any open source project
    • Great contributors adding features right now...
    • Time to make Horizon "sexy"!
