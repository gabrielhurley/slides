Horizon

April 2013 Project Update

Gabriel Hurley / @gabrielhurley

Why Horizon?

Horizon is necessary for the growth and success of the community.

  • Fastest way for someone new to OpenStack to understand it.
  • Easiest way to understand the breadth what OpenStack is capable of.
  • Crucial tool for driving adoption.
  • Serves as "the face of OpenStack".

What happened in the last 6 months?

Grizzly

Features

Image upload, Instance migration, Quantum routers, load balancers, network topology visualization, VNIC ordering, flavor "extra specs", simplified floating IP management, simplified security group rule editing, image organization, API access info all in one place, better "unauthorized" API error handling, icons, and more!

Contributors

  • 20+ companies
  • 50 contributors
  • 22 blueprints
  • 130 bugs fixed
  • 200 changesets

Community

  • 4 new core developers from RedHat, NEC, and University of Melbourne
  • ~30 new contributors
  • Weekly IRC meeting
  • New documentation
  • Moved from one company dominating the project to the community building together

Compatibility

Completely backwards-compatible with Folsom.

Many people run Horizon master against their older OpenStack installations.

Grizzly is the best release of Horizon yet!
